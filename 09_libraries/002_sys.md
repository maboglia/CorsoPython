# Modulo `sys`: Il controllo dell'Interprete

Mentre `os` gestisce il computer, `sys` gestisce l'ambiente di Python stesso.

* **L'uso dei parametri (`sys.argv`):** È fondamentale per creare script che accettano input da terminale (es: `python script.py file_da_leggere.txt`).
* **Gestione dei percorsi (`sys.path`):** Spiega che se Python "non trova un modulo", è perché quel percorso non è in questa lista.
* **Esempio didattico:**

```python
import sys
print(f"Versione di Python in uso: {sys.version}")
print(f"Argomenti passati allo script: {sys.argv}")

```

# 📌 Libreria `sys` – Scheda Completa

Serve per interagire con l’ambiente Python:

* argomenti CLI
* uscita programma
* path moduli
* stdin/stdout

```python
import sys
```

---

## 1) Argomenti da riga di comando (`sys.argv`)

Esecuzione:

```bash
python script.py Mario 20
```

Script:

```python
import sys

print(sys.argv)
```

📌 Output:

```python
['script.py', 'Mario', '20']
```

---

## 2) Gestione parametri CLI

```python
import sys

if len(sys.argv) != 3:
    print("Uso: python script.py nome eta")
    sys.exit(1)

nome = sys.argv[1]
eta = int(sys.argv[2])

print(nome, eta)
```

---

## 3) Terminare programma (`sys.exit`)

```python
import sys

sys.exit(0)   # terminazione normale
sys.exit(1)   # errore
```

---

## 4) Input e output standard

### Output su console

```python
import sys

sys.stdout.write("Ciao\n")
```

### Error output

```python
sys.stderr.write("Errore!\n")
```

---

## 5) Informazioni su Python

```python
import sys

print(sys.version)
print(sys.platform)
```

---

## 6) Percorso moduli Python (`sys.path`)

```python
import sys

print(sys.path)
```

Aggiungere un percorso:

```python
sys.path.append("C:/miei_moduli")
```

---

## 7) Dimensione oggetto in memoria

```python
import sys

x = [1,2,3]
print(sys.getsizeof(x))
```

---

## 8) Best practice

✅ usare `sys.argv` per script CLI
✅ usare `sys.exit()` per errori controllati
⚠️ per parsing serio meglio `argparse`
