# **11 – Iterables**

Un **iterable** è un oggetto su cui è possibile **iterare**, cioè visitare i suoi elementi **uno alla volta** usando un ciclo `for`.
In Python, il concetto di iterabile è fondamentale per comprendere cicli, comprehension e generatori.

---

## **Cosa significa iterabile**

Un oggetto è **iterabile** se:

* può essere usato in un ciclo `for`
* fornisce un **iteratore** tramite la funzione `iter()`

```python
numeri = [1, 2, 3]

for n in numeri:
    print(n)
```

---

## **Esempi di iterabili comuni**

* `list`
* `tuple`
* `set`
* `dict`
* `string`
* `range`
* `file`
* `generator`

```python
for c in "Python":
    print(c)
```

---

## **Iterabile vs Iteratore**

* **Iterabile**: oggetto che può essere iterato
* **Iteratore**: oggetto che restituisce gli elementi uno alla volta

```python
numeri = [1, 2, 3]
it = iter(numeri)

print(next(it))  # 1
print(next(it))  # 2
print(next(it))  # 3
```

---

## **Uso implicito degli iterabili**

Python usa gli iterabili automaticamente in:

* cicli `for`
* list/dict/set comprehension
* funzioni come `sum()`, `max()`, `min()`, `any()`, `all()`

```python
print(sum([1, 2, 3]))
```

---

## **Dizionari come iterabili**

Per default, iterare su un dizionario restituisce le **chiavi**:

```python
studente = {"nome": "Anna", "eta": 20}

for k in studente:
    print(k)
```

---

## **Controllare se un oggetto è iterabile**

```python
from collections.abc import Iterable

print(isinstance([1, 2, 3], Iterable))  # True
print(isinstance(10, Iterable))         # False
```

---

## **Iterabili personalizzati**

Un oggetto è iterabile se implementa:

* `__iter__()`
* oppure `__getitem__()`

> Questo concetto è alla base dei **generatori** e delle **classi personalizzate**.

---

## **In sintesi**

* Tutti i cicli `for` lavorano su iterabili
* Gli iterabili sono ovunque in Python
* Capirli rende il codice più idiomatico
